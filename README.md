项目详细说明：游戏卡牌识别系统
一、功能概述
这是一个基于 OpenCV 传统计算机视觉方案的自动卡牌检测系统。核心功能是：
1.自动判断一张图片是否为"游戏卡牌"（如游戏王、万智牌、宝可梦卡牌等）
2.识别为卡牌：输出置信度和边界框，可选择上传到后端 API
3.非卡牌：直接跳过，不做处理
4.批量处理：支持单张图片或整个目录的批量扫描
5.结果导出：可将检测结果导出为 JSON 文件

二、实现原理
2.1 多维度加权评分架构
系统不使用深度学习模型，而是通过 5 个视觉维度对图片打分，加权求和得到最终置信度（0~1），超过阈值（默认 0.55）即判定为卡牌。
 维度	  权重	   实现方法
轮廓矩形	30%	Canny/Sobel/Otsu 三种边缘检测取最优 → 找最大凸四边形轮廓 → 验证角度接近90°、非纯白背景
宽高比	  25%	基于边界框计算长边/短边比值，在 1.3~1.85 之间给分，理想值 1.55
面积占比	15%	卡牌边界框占画面面积比例，5%~100% 有效，理想值 35%（满幅卡牌特殊处理给 0.75 分）
角点分布	15%	Harris 角点检测，要求 4 个角点分布在画面 4 个象限
边缘纹理	15%	Canny 边缘密度检测，排除纯色图片（密度<0.02）和过度复杂图片（密度>0.40）
2.2 轮廓检测的三重策略（card_detector.py）
这是最关键也最复杂的部分。代码采用了三种边缘检测方法取最优的策略：
Canny 边缘检测 → 强对比度边界（卡牌有明显边框时有效）
Otsu 二值化    → 柔和边界（纯色边框、低对比度时有效）
Sobel 梯度     → 梯度变化（满幅卡牌、边界不明显时有效）

对每种边缘图，执行：
找轮廓 → 按面积排序取前 5 个最大轮廓
多边形近似（approxPolyDP）→ 如果恰好是 4 个顶点的凸四边形
角度验证 → 4 个内角应接近 90°
填充率验证 → 轮廓面积与矩形面积的比例
白背景排除 → 检测矩形四边是否为纯白色（排除截图/UI 元素）
回退到 minAreaRect → 如果顶点数 > 4 但旋转矩形填充率高且面积占比合理
2.3 白背景排除（card_detector.py:190-204）
这是一个关键的反误判机制。通过检测矩形四边 8 像素范围内的均值和标准差：
纯白色背景（均值 > 230 且标准差 < 10）→ 排除（通常是截图、UI 元素）
真实卡牌四边有颜色变化 → 通过
2.4 后端上传（backend_client.py）
使用 requests.Session 保持连接
multipart/form-data 格式上传
附带 confidence、bbox_x/y/w/h 等参数
失败自动重试 3 次
支持自定义请求头（如认证 Token）
2.5 命令行入口（main.py）
--no-upload：只检测不上传
--threshold：调整检测阈值
--output：导出 JSON 结果
--backend-url：自定义后端地址
--header：添加认证请求头

三、开发过程中遇到的问题与解决方案
问题 1：单一边缘检测无法覆盖所有卡牌类型
现象：有些卡牌边框对比度低（纯色卡背），Canny 检测不到轮廓；有些满幅卡牌（图案铺满整个画面），边界与背景融合。
解决方案：在 _score_contour 方法中引入三种边缘检测并行取最优的策略（card_detector.py:82-88）：
Canny：强边缘
Otsu 二值化：柔和边界
Sobel 梯度：满幅卡牌的梯度变化

问题 2：误判截图/UI 元素为卡牌
现象：微信截图、网页截图中的白色矩形框被误识别为卡牌。
解决方案：增加 _is_white_background 方法（card_detector.py:190-204），检测矩形四边的像素均值和标准差，排除纯白色背景的元素。

问题 3：宽高比阈值过松导致误判
现象：细长的物体（如笔、湿巾包装）也被识别为卡牌。
解决方案：将宽高比范围从最初的 1.2~2.0 收紧到 1.3~1.85（card_detector.py:29），理想值定为 1.55（接近标准游戏卡牌比例）。

问题 4：满幅卡牌面积占比归零
现象：当卡牌占满整个画面（bbox 几乎等于图片尺寸），_score_by_ratio 函数因为面积比 > 1.0 超出上限而返回 0 分。
解决方案：在面积占比评分中增加特判（card_detector.py:56-57），当 area_ratio > 0.90 时直接给 0.75 固定高分。

问题 5：旋转卡牌的矩形检测
现象：卡牌倾斜放置时，标准 boundingRect 无法贴合轮廓。
解决方案：当多边形近似顶点数 > 4 时，回退使用 cv2.minAreaRect 旋转矩形检测（card_detector.py:150-164），但设置了更严格的条件：填充率 > 0.80 且面积占比 > 5% 才接受。

问题 6：多边形近似精度调参
现象：approxPolyDP 的 epsilon 系数太大会把不规则形状也近似为四边形，太小则无法将真实卡牌的四边形识别出来。
解决方案：经过多次实验，将系数定为 0.02 × 周长（card_detector.py:129），配合角度验证（4 个角接近 90°）来过滤假阳性。
